## First Maker Comment (pin this at 00:05 PT)

```
Hey PH! Maker here.

I kept seeing the same bugs in repos people posted on Twitter and HN:
API keys committed to GitHub, Supabase service_role tokens in client
components, Next.js routes with no auth at all. Every AI tool ships
with its own specific "forgot" pattern.

vibe-hardening is a CLI with 30+ rules tuned specifically for code
generated by v0, Lovable, Bolt, Cursor, Claude Code, Replit Agent,
Windsurf and Devin. It fingerprints which tool wrote the code and
weights rules accordingly.

Try:
  npx vibe-hardening scan

in any repo. Exit code 1 when critical or high findings appear, so
it drops into CI. Produces HTML reports, JSON for CI, a 0-100 score,
and an SVG badge for your README.

MIT, no account, nothing to sign up for.

I built this over four days. The first real run on my own project
found a missing-auth bug in a cron endpoint I would not have
caught until someone exploited it. If you scan a repo and it finds
something silly or misses something obvious, please tell me — the
rules are still evolving and real data beats synthetic fixtures.

Thanks for checking it out 🫡
```

## Launch-day reply templates

**"Does this replace Snyk / Semgrep?"**
```
No — those are better for mature teams. This is for the indie / vibe
coder who'd never install a security scanner otherwise. Complement,
not replacement.
```

**"What about Python / Go / Rust?"**
```
v0.1 is JS/TS focused. Python is next on the roadmap because
Cursor-written Python apps have the same problems. Go if requested
enough.
```

**"Can I self-host?"**
```
CLI is fully local — no telemetry, no phone-home, nothing leaves your
machine. OSV and npm registry calls go direct to public APIs from
your machine. Disable with --offline.
```
