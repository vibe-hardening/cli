## Launch post (publish 2026-05-13 morning)

```
Most AI coding tools ship impressive demos but leave production
security to the human on the other side.

After reviewing 50+ repos generated by v0, Lovable, Bolt, Cursor,
Claude Code and Windsurf, the pattern was undeniable:

→ 36% had committed API keys
→ 28% shipped Supabase service_role tokens to the browser
→ 45% had at least one API route with no authentication check

I built vibe-hardening to close the gap between "code generated in
30 seconds" and "code safe to ship."

One command. No configuration. Open source.

  npx vibe-hardening scan

The scanner understands which AI tool produced the code (via
fingerprinting) and applies rules tuned for that tool's common
oversights. It checks for:

• Leaked API keys (OpenAI, Anthropic, Stripe, GitHub, AWS, Supabase)
• Missing authentication on Next.js API routes (AST-based)
• Supabase tables shipped without Row Level Security
• Injection vectors (SQL, NoSQL, shell, path traversal, XSS)
• CORS misconfigurations and SSRF
• Known CVEs via OSV.dev
• LLM-hallucinated package names (slopsquatting)

Free and MIT on GitHub. Roadmap includes a team dashboard and
GitHub App integration.

For engineering leads: this is a quick diagnostic you can run
against any vibe-coded proof of concept before it becomes a
production commitment. For indie builders: it's a safety net you
can add to CI with one npx call.

vibe-hardening.io | github.com/vibe-hardening/cli

#AppSec #Security #OpenSource #DevTools #AICodeReview
```

## Follow-up post (1 week after launch, different angle)

```
Security for AI-coded applications is a governance problem, not a
scanning problem.

When an LLM generates 80% of your codebase in 30 minutes, code
review stops being the bottleneck. Nobody reads 2000 lines of Next.js
route handlers carefully. The review surface has to move earlier —
to the generation step, or later — to an automated policy gate.

vibe-hardening takes the automated-gate approach. CI-friendly exit
codes, JSON output for downstream tooling, inline suppression
directives for the rare legitimate exceptions.

But the deeper fix is upstream: AI tools that refuse to emit an API
route without an auth check, or a service_role key in a client
component. We're at the "seatbelt in the 1970s" stage of AI-coded
app safety — the infrastructure is there, adoption isn't.

vibe-hardening.io
```
