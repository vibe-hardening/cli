## 1 / Product context

**vibe-hardening** is a one-command security scanner for AI-generated code.

- Target user: "vibe coders" — indie hackers / non-security devs who ship apps generated by v0, Lovable, Bolt, Cursor, Claude Code, Replit Agent, Windsurf
- Core value prop: `npx vibe-hardening scan` → 5-second security audit
- What it catches: leaked API keys, Supabase RLS missing, NEXT_PUBLIC_* secrets, missing auth checks, CORS wildcards, SSRF, SQL injection, LLM-hallucinated packages, dependency CVEs
- Tagline: **Vibe coded. Vibe hardened.**
- Domain: vibe-hardening.io
- Source: github.com/vibe-hardening/cli
- License: MIT (CLI) + Pro SaaS tier (dashboard, team features)
- Launch: 2026-05-13 on Product Hunt

## 2 / Design brief

### Chosen archetype
**Tactical Telemetry / CRT Terminal (Dark mode)** from the brutalist
design spec. Declassified military database / aerospace HUD aesthetic
merged with Swiss industrial print typography.

### Palette (locked)
- Background: `#0A0A0A` (deactivated CRT)
- Foreground: `#EAEAEA` (white phosphor)
- Accent: `#FF2A2A` (aviation red — alerts / accents only)
- Green: `#4AF626` — used ONLY for status tick and terminal cursor, never general text
- Line: `#2A2A2A` (hairline dividers)
- Dim: `#6A6A6A` (metadata text)

### Type stack (locked)
- Macro display: **Archivo Black** (weight 900), `clamp(3.5rem, 14vw, 15rem)`, tracking `-0.05em`, line-height `0.82`, uppercase
- CJK macro: **Noto Sans TC** (weight 900) — used on /zh page only
- Micro / data / UI: **JetBrains Mono**, 10-14px, tracking `+0.08em`, uppercase

### Rules
- No border-radius, all 90° corners
- No gradients, no soft shadows, no modern glassmorphism
- Visible 1px grid dividers between zones
- ASCII framing: `[ LABEL ]`, `>>> EXECUTE`, crosshair `+` markers
- Industrial markers: `®`, `™`, barcode stripes, unit IDs (`VH-001`, `REV 0.0.1`)
- CRT scanlines overlay + low-opacity SVG noise texture

## 3 / Current sections

1. **Top strip** — `VH-001 / REV 0.0.1 / ● ONLINE / MIT` in monospace
2. **Nav** — logo + 3 links (SOURCE ↗ / WAITLIST / EN⇄中文 switcher)
3. **Hero** — `VIBE CODED. / VIBE HARDENED.®` massive + `npx vibe-hardening scan` CTA box with blinking green cursor
4. **Features** — 3 units with heading + ID codes only, no body copy (`001 / DB-01 → AI-AWARE RULES` etc.)
5. **Terminal** — simulated scan output with CRITICAL/HIGH findings + SCORE line
6. **Waitlist** — `SHIP HARDENED.` + email form
7. **Footer** — copyright + 3 links + giant watermark

Full page weight: ~52KB HTML, 105KB JS (static prerender).

## 4 / Two locales

- `/` English (default) — brand + all body copy in English
- `/zh` Traditional Chinese — body copy translated, but **hero tagline stays English as brand**, all ASCII meta tags (VH-001 / REV / MIT / H-01 / S-02 / T-03 / D-04 / `[CRITICAL]`) stay English as "international technical codes"

## 5 / What I want from Claude Design

### A. Critical review
Act as a senior design director at a premium indie-dev-tool agency
(think vercel.com, linear.app, railway.app, retool.com, resend.com
level of visual discipline). Critique the current design on:

- Typography hierarchy — does the scale jump work, or do any headings fight each other?
- Information density — is there dead weight, or are zones too sparse?
- Visual weight — does the red accent earn its placement?
- Brand coherence — does "vibe coded" (playful) coexist believably with "vibe hardened" (military)?
- Telemetry authenticity — do the meta tags / codes read as "real engineered product" or "dress-up"?
- Mobile behaviour — what breaks on narrow widths?
- CJK render — does the bilingual design hold together or does /zh look like an afterthought?

Be harsh. Ship the specific line numbers / class names when possible.

### B. Three visual variants (mockups)

Produce three full-page visual mockups as alternatives to the current
version, each ~3 sections (hero / features / terminal or waitlist):

**Variant A — Swiss editorial**
Lighter density, larger negative space, hero as single line with a
serif pull-quote, grid still rigid but "calmer".

**Variant B — Military HUD maximalist**
MORE telemetry: coordinates, radar sweeps, classification banners,

**Variant C — Linear-esque minimal**
Strip all ornament. Hero / one stat / terminal / CTA. Keep brutalist
palette and type scale but remove ASCII decoration entirely. Test if
discipline alone carries the look.

### C. Final recommendation
Pick one of (current / A / B / C) or recommend a fusion. Give a
