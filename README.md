# vibe-hardening

> **Vibe coded. Vibe hardened.**
>
> One-command security scanner for AI-generated code.

```bash
npx vibe-hardening scan
```

Live at [vibe-hardening.io](https://vibe-hardening.io).

## What it catches

Designed for repos generated by **v0 / Lovable / Bolt / Cursor / Claude Code / Replit Agent / Windsurf / Devin**.

| Category | Examples |
|----------|----------|
| Leaked secrets | OpenAI `sk-proj-`, Anthropic `sk-ant-`, Stripe `sk_live_`, GitHub PAT, AWS keys, Supabase `service_role`, DB URLs, Slack tokens, JWT signing keys |
| Supabase config | Tables created without `enable row level security` |
| Env misuse | `NEXT_PUBLIC_*SECRET` / `NEXT_PUBLIC_*SERVICE_ROLE` — variables leaking private values into client bundles |
| Missing auth | Next.js App Router route handlers (`app/api/**/route.ts`) and Pages API handlers that export HTTP methods without any detectable auth call (AST-based, understands `supabase.auth.getUser()` / `getServerSession()` / `auth()` / `requireAuth()` / HOC wrappers) |

## Usage

```bash
# Scan current directory
npx vibe-hardening scan

# Scan a specific folder
npx vibe-hardening scan ./my-project

# JSON for CI
npx vibe-hardening scan --format json --output report.json

# Only report high-severity findings
npx vibe-hardening scan --severity high
```

Exits with code `1` when any critical or high finding is reported (CI-friendly).
